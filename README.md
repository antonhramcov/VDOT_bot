# VDOT Telegram Bot

Telegram bot на aiogram, который принимает результат бега, отвечает рассчитанным
VDOT и показывает ориентировочные тренировочные темпы.

## Формат сообщений

```text
5 км за 24:30
10 км за 52:10
полумарафон за 1:55:00
```

Также поддерживаются `km`, `k`, `марафон`, `marathon`, `half`, `half marathon`,
`hm`.

## Команды

```text
/start
/my_vdot
/trains
/language
```

Команда `/my_vdot` показывает сохраненный VDOT текущего пользователя. Если VDOT
еще не сохранен, бот попросит прислать результат забега.

Команда `/trains` показывает текущие тренировочные темпы по сохраненному VDOT.
Команда `/language` открывает выбор языка.

При старте бот регистрирует команды через Telegram API, чтобы они отображались
в меню команд у пользователя.

В стартовом сообщении есть кнопка `Language`. Она открывает выбор языка:
`🇷🇺 Русский` или `🇬🇧 English`. После выбора бот заменяет сообщение на стартовое
на выбранном языке. По умолчанию для новых пользователей используется английский.

Когда пользователь присылает первый результат, бот сохраняет VDOT и предлагает
рассчитать тренировочные темпы кнопкой. Если новый результат отличается от
сохраненного VDOT, бот сообщает, вырос или снизился показатель, и предлагает
кнопками выбрать: обновить сохраненный VDOT или оставить предыдущее значение.

После расчета тренировочных темпов бот показывает кнопки по типам тренировок.
По нажатию на кнопку сообщение заменяется описанием, как выполнять выбранную
тренировку. Кнопка возврата открывает список тренировочных темпов обратно.

## Запуск

Перед запуском добавьте токен и строку подключения к PostgreSQL в окружение:

```bash
export BOT_TOKEN=123456:telegram-token
export DATABASE_URL=postgresql://gen_user:password@192.168.0.5:5432/default_db
docker compose up --build
```

Или создайте локальный `.env`:

```env
BOT_TOKEN=123456:telegram-token
DATABASE_URL=postgresql://gen_user:password@192.168.0.5:5432/default_db
```

Затем:

```bash
docker compose up --build
```

## Расчет VDOT

Используется формула Daniels/Gilbert:

```text
VO2 = -4.60 + 0.182258 * v + 0.000104 * v^2
fraction = 0.8 + 0.1894393 * e^(-0.012778 * t) + 0.2989558 * e^(-0.1932605 * t)
VDOT = VO2 / fraction
```

где `v` — скорость в метрах в минуту, `t` — время в минутах.

## База данных

Бот подключается к PostgreSQL через переменную окружения `DATABASE_URL`.
При старте он создает таблицы `bot_users` и `user_vdots`, если их еще нет.
В `bot_users` хранится язык пользователя, по умолчанию `en`. После каждого
успешного расчета VDOT бот сохраняет последнюю запись пользователя по его
Telegram ID в `user_vdots`. Локальный `.env` добавлен в `.gitignore`, поэтому
реальные доступы к базе не попадут в репозиторий.

Тексты интерфейса вынесены в `texts.py`. Чтобы добавить новый язык, нужно
добавить код языка, флаг, native name и набор строк в словарь `LANGUAGES`.

## Тренировочные темпы

После расчета VDOT бот дополнительно показывает ориентировочные темпы:

```text
Easy / Легкий бег
Marathon / Марафонский темп
Threshold / Пороговая тренировка
Interval / Интервалы
Repetition / Повторения
```

Темпы считаются из VDOT через обратную VO2-формулу и интенсивности тренировочных
зон. Это не копия таблиц Daniels, а компактная расчетная модель для практических
ориентиров.
